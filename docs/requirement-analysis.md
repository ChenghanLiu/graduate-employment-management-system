# 需求分析

## 项目定位

构建一个基于 Spring Boot 的毕业生就业管理系统，采用单体架构，前后端分离部署，围绕毕业生求职、企业招聘、就业审核与公告发布等核心业务逐步迭代。

## 本轮范围

- 明确系统功能模块分区与端侧边界
- 明确按角色划分的菜单结构设计
- 在现有 RBAC 基础上补充核心业务数据模型
- 在后端预留业务模块包结构
- 更新需求、系统设计、数据库设计文档

## 功能分区

### 管理端

面向系统管理员、就业管理老师/辅导员，负责全局配置与业务审核，重点模块如下：

- 系统首页：查看待审核事项、就业进度、公告概览
- 学生管理：查看学生档案、就业状态、简历完整度
- 企业管理：查看企业资料、企业认证审核结果
- 招聘管理：查看岗位发布情况、岗位上下线状态、投递记录
- 就业管理：审核就业去向、维护就业附件、跟踪就业落实情况
- 公告管理：发布和下线公告
- 系统管理：用户、角色、基础字典等后续扩展入口

### 学生/企业端

面向学生与企业用户，聚焦自助维护和业务提交，不承担后台审核职责。

- 学生侧：维护个人档案、编辑简历、投递岗位、提交就业信息、查看公告
- 企业侧：维护企业资料、发布岗位、处理学生投递、查看公告

## 角色边界

- `ADMIN`：系统管理员，负责系统配置、全量数据查看和业务管理
- `COUNSELOR`：就业管理老师/辅导员，负责学生就业审核、企业审核、业务跟踪
- `STUDENT`：学生用户，负责个人资料、简历、岗位投递与就业信息填报
- `ENTERPRISE`：企业用户，负责企业资料维护、岗位发布与投递处理

## 本轮非目标

- 前端业务页面开发
- 完整 CRUD 接口与服务实现
- 复杂权限树、审批流引擎、消息通知
- 报表统计、数据大屏、导入导出
- 多租户、微服务拆分、复杂中间件接入
